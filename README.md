# Open Structure Library (OSL)

> **分野の壁を超え、問題の構造を共有する / Sharing problem structures beyond disciplinary boundaries.**

Open Structure Library (OSL) は、学術論文や専門的な思考プロセスから抽出された「問題の骨格（構造）」を蓄積し、共有するためのオープンデータプロジェクトです。

本プロジェクトの目的は、特定の専門用語（語彙）に依存しない抽象的な構造を記述することで、異なる分野間での**構造的同型性（Isomorphism）**を発見し、知見の転写を促進することにあります。

---

## 🌌 ビジョン
私たちは、現代の複雑な課題解決のヒントは、しばしば全く異なる学問分野に既に存在していると信じています。
例えば、組織の硬直化の問題は、物理学の相転移や生態系のレジリエンスの崩壊と同型の構造を持っているかもしれません。
`.isom` 形式で記述された構造を共有することで、AIと人間が協力してこれらの「知のミッシングリンク」を繋ぎ直すインフラを構築します。

---

## 🛠 ファイル形式: `.isom` (Isomorphism DSL)

本ライブラリでは、因果関係、制約条件、インセンティブ構造などを記述するために、独自に定義された `.isom` (Isomorphism DSL) 形式を使用します。

### 特徴
- **脱表現化**: 著者の具体的な言い回しを、変数と論理的関係に置き換えます。
- **4つのオントロジー**: すべての要素を `Agent`, `Event`, `Resource`, `Intentional Moment` の4つの型に強制し、構造比較を容易にします。
- **因果の極性**: 関係性が「促進 (+)」か「抑制 (-)」かを明示します。



---

## 📁 ディレクトリ構成

本リポジトリは、その出自（プロバンス）に基づいて厳格に管理されています。

- `/library/by-domain/`: 既存の学問体系（AI/ML, 生態学, 心理学など）ごとの構造データ。
- `/library/by-pattern/`: 「評価のギャップ」「自己強化ループ」などの構造パターン別のインデックス。
- `/ontology/`: 使用される型とDSLの文法仕様。

## 🤝 貢献方法 (How to Contribute)

新しい構造の提案を歓迎します！

1. **テンプレートの利用**: `templates/submission_template.isom` をコピーして作成してください。
2. **文献情報の記載**: 論文由来の場合は、必ずDOIやURLなどのソース情報をYAMLヘッダーに含めてください。
3. **バリデーション**: プルリクエストを送信すると、自動CI（GitHub Actions）がスキーマと構文のチェックを行います。

※ 独自の未整理な思考ログやアイデアについては、姉妹プロジェクトである `open-structure-sandbox` への投稿を推奨しています。
